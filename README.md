# StockV2.1 - AI Stock Analysis

Based on [Daily Stock Analysis](https://github.com/cbh2001/daily_stock_analysis).

## New Features in V2.1
- **Multi-Tab Interface**: Organized into Dashboard, Strategy, and News tabs for better usability.
- **API Key Memory**: Automatically saves API keys to `.env` file, persisting them across sessions and refreshes.
- **Multi-Model Support (PK Mode)**: Add and manage multiple AI models (OpenAI, DeepSeek, Kimi, etc.) simultaneously. Run analysis on all active models to compare (PK) their predictions and logic side-by-side.
- **API Validation**: Built-in validation to check API key validity and connectivity before use.
- **Event-Driven Strategy**: "One Week Ahead" market outlook based on news events (e.g., Geopolitical -> Defense, AI Updates -> Tech, Earnings -> Market moves).
- **Real-time News**: Integrated financial news feed using AkShare for up-to-date market information.
- **UI Optimization**: Enhanced layout and user experience.

## Features
- **AI Decision Dashboard**: Daily conclusions, analysis, and action plans generated by LLM.
- **Data Fetching**: Real-time A-share/HK-share data via AkShare.
- **Multi-channel Push**: Email notifications (extensible).
- **Financial Knowledge Base**: Self-updating repository of financial concepts with search and manual entry support.
- **WebUI**: Interactive dashboard using Streamlit.

## Setup

1. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

2. Configure environment:
   Copy `.env.example` to `.env` and fill in your API keys.
   *Note: In V2.1, you can also input and save API keys directly in the WebUI sidebar.*

3. Run WebUI:
   ```bash
   streamlit run app.py
   ```

4. Run Daily Job (CLI):
   ```bash
   python main.py
   ```

## 📦 离线包安装部署手册 (Offline Installation Guide)

本指南适用于无法访问外网或使用离线压缩包 (`StockV2.2_Dist_YYYYMMDD.zip`) 进行部署的场景。

### 1. 服务器准备 (Server Preparation)
- **操作系统**: Ubuntu 20.04+ / CentOS 7+ / Windows Server
- **Python环境**: Python 3.10 或更高版本
- **基础工具**: `unzip` (解压工具)

### 2. 安装步骤 (Installation Steps)

#### 步骤 1: 解压文件
将压缩包上传至服务器目录 (例如 `/home/host/StockV2.1`) 并解压：
```bash
unzip StockV2.2_Dist_20260205.zip -d StockV2.1
cd StockV2.1
```

#### 步骤 2: 创建虚拟环境 (推荐)
为了避免污染系统环境，建议使用虚拟环境：
```bash
# 创建虚拟环境
python3 -m venv venv

# 激活环境 (Linux/Mac)
source venv/bin/activate

# 激活环境 (Windows)
# venv\Scripts\activate
```

#### 步骤 3: 安装依赖
```bash
pip install -r requirements.txt
```
*注：如果是完全离线环境，需要预先下载好 whl 依赖包并使用 `pip install --no-index --find-links=./wheels -r requirements.txt` 安装。*

#### 步骤 4: 配置环境
复制示例配置文件并填入您的 API Key：
```bash
cp .env.example .env
nano .env
```
或者在首次启动后通过 Web 界面 (Model Management) 添加 Key。

#### 步骤 5: 启动服务
**前台运行 (测试用)**:
```bash
streamlit run app.py --server.port 8080
```

**后台稳定运行 (生产环境)**:
```bash
nohup streamlit run app.py --server.port 8080 --server.address 0.0.0.0 > app.log 2>&1 &
```

### 3. 验证安装
执行自带的检查清单脚本，确保所有模块正常：
```bash
python checklist.py
```
如果看到 `🎉 Ready for Deployment!`，说明安装成功。

### 4. 常见问题排查
- **sqlite3.OperationalError: no such table: predictions**:
  - 原因：数据库初始化未执行。

- **端口被占用**: 检查是否有其他服务占用 8080 端口，或者旧的 streamlit 进程未被杀掉。
- **依赖安装失败**: 检查网络连接，或尝试手动运行 `pip install -r requirements.txt --break-system-packages`。
- **页面打不开**: 检查服务器防火墙是否放行 8080 端口。

## 📦 离线安装与分发 (Offline Installation & Distribution)

本节说明如何打包项目以便在无网络或隔离环境中安装，以及系统配置要求。

### 1. 系统配置要求 (System Requirements)
- **操作系统**: Linux (Ubuntu 20.04+, CentOS 7+), Windows 10/11, macOS.
- **Python 版本**: Python 3.8 或更高版本.
- **内存 (RAM)**: 建议 2GB 以上 (Pandas 和 Streamlit 运行时占用).
- **硬盘空间**: 至少 500MB 可用空间.
- **网络**: 
  - 必须: 能访问 LLM API 服务商 (OpenAI/DeepSeek/Gemini).
  - 必须: 能访问 AkShare 数据源 (通常为国内网络畅通即可).

### 2. 获取完整安装包
运行项目根目录下的打包脚本，生成可分发的压缩包：
```bash
python package_dist.py
```
该命令会在同级目录生成 `StockV2.1_Dist.zip`，包含所有源代码和依赖说明，但不包含敏感配置 (.env) 和 临时数据 (.db)。

### 3. 安装步骤 (Installation Steps)
1. **解压文件**: 将 `StockV2.1_Dist.zip` 解压到目标目录。
2. **安装依赖**:
   ```bash
   pip install -r requirements.txt
   ```
   *(注: 如果是完全离线环境，需提前下载好 whl 包)*
3. **配置环境**:
   - 复制 `.env.example` 为 `.env`。
   - 填入 API Key (或启动后在网页端配置)。
4. **启动服务**:
   ```bash
   streamlit run app.py
   ```
